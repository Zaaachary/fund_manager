# 测试样本 04: 高频操作与数据一致性

## 测试目的
验证系统在高频操作下的稳定性，包括快速连续操作、并发操作模拟、数据一致性保持。

## 测试数据
- 基金名称: "高频测试基金"
- 投资者: 5位高频交易者

## 测试步骤

### Phase 1: 快速连续申购赎回
1. **单投资者高频操作**
   - 投资者A在1小时内执行以下操作:
     - 申购 ¥10,000
     - 5分钟后赎回 5,000份
     - 10分钟后申购 ¥20,000
     - 5分钟后赎回 ¥15,000（按金额）
     - 10分钟后转账给B 3,000份
   - **验证**: 每次操作后份额和资产计算正确

2. **净值更新穿插**
   - 在上述操作中穿插3次净值更新:
     - NAV 1.0 → 1.1 → 1.05 → 1.2
   - **验证**: 净值变化时申购/赎回计算正确

### Phase 2: 多投资者并发操作模拟
3. **同一时间点多投资者操作**
   - 模拟"同一时刻"（秒内）多投资者操作:
     - 投资者A: 申购 ¥50,000
     - 投资者B: 赎回 10,000份
     - 投资者C: 转账给D 5,000份
     - 投资者D: 申购 ¥30,000
     - 投资者E: 赎回 ¥20,000（按金额）
   - **验证**: 所有操作按顺序处理，最终数据一致
   - 验证基金总份额变化 = 各投资者份额变化总和

### Phase 3: 净值高频更新
4. **一天内多次净值更新**
   - 模拟交易日高频更新:
     - 开盘: NAV 1.0
     - 10:00更新: NAV 1.02
     - 11:30更新: NAV 1.05
     - 13:00更新: NAV 1.03
     - 14:30更新: NAV 1.08
     - 收盘: NAV 1.10
   - 每次更新间穿插投资者操作
   - **验证**: 历史记录完整，无遗漏

5. **净值大幅波动更新**
   - 快速涨跌场景:
     - NAV 1.0 → 1.5 (+50%)
     - NAV 1.5 → 0.8 (-47%)
     - NAV 0.8 → 2.0 (+150%)
   - **验证**: 系统正确处理大幅波动

### Phase 4: 数据一致性验证
6. **总份额守恒验证**
   - 公式: 基金总份额 = Σ(所有投资者份额)
   - 随机检查10个时间点验证等式成立
   - 转账操作后特别验证（总份额应不变）

7. **总资产守恒验证**
   - 公式: 基金总资产 = Σ(所有投资者资产)
   - 申购/赎回时资产流入流出验证
   - 净值更新时重新计算验证

8. **操作记录完整性验证**
   - 所有操作都有对应记录
   - 操作前后数值正确记录
   - 时间戳顺序正确

### Phase 5: 边界值测试
9. **极小金额高频操作**
   - 100次申购，每次¥0.01
   - 验证精度保持（6位小数）
   - 验证无精度丢失累积

10. **极大金额操作**
    - 单笔申购 ¥100,000,000
    - 验证系统正常处理大额

11. **零值操作拒绝**
    - 尝试申购¥0
    - 尝试赎回0份
    - **预期**: 系统拒绝并提示错误

### Phase 6: 复杂场景组合
12. **模拟真实交易日**
    - 创建场景:
      - 开盘: 5位投资者各申购不同金额
      - 盘中: 多次净值更新，部分投资者赎回
      - 午盘: 新增2位投资者，转账操作
      - 收盘: 最终净值更新，全部赎回
    - **验证**: 全天数据一致性

13. ** stress test**
    - 100次随机操作序列:
      - 随机选择投资者
      - 随机选择操作类型（申购/赎回/转账）
      - 随机金额
    - **验证**: 系统稳定，无崩溃

## 关键验证点
- [ ] 高频操作下系统响应正常
- [ ] 快速连续操作数据一致
- [ ] 多投资者操作后总份额守恒
- [ ] 多投资者操作后总资产守恒
- [ ] 一天内多次净值更新记录完整
- [ ] 极小金额操作精度保持
- [ ] 极大金额操作正常处理
- [ ] 零值/负值操作被拒绝
- [ ] 复杂场景后数据一致性保持

## 涉及后端能力
- 全部核心功能
- 数据一致性校验
- 并发操作处理（顺序执行）
