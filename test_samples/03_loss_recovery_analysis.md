# 测试样本 03: 亏损场景与回本分析

## 测试目的
验证基金下跌时的投资者盈亏计算、亏损状态下的赎回、以及回本点分析。

## 测试数据
- 基金名称: "波动测试基金"
- 投资者: 张三(激进), 李四(稳健), 王五(抄底)

## 测试步骤

### Phase 1: 建仓与上涨
1. **基金创建与建仓**
   - 创建基金，添加三位投资者
   - 张三申购: ¥200,000 → 200,000份
   - 李四申购: ¥150,000 → 150,000份
   - 王五申购: ¥100,000 → 100,000份
   - 总份额: 450,000份

2. **第一波上涨50%**
   - 更新总资产到¥675,000
   - NAV = 675,000/450,000 = 1.5
   - **各投资者盈利**:
     - 张三: 200,000 × 1.5 - 200,000 = **+¥100,000**
     - 李四: 150,000 × 1.5 - 150,000 = **+¥75,000**
     - 王五: 100,000 × 1.5 - 100,000 = **+¥50,000**

### Phase 2: 大幅下跌
3. **第一波下跌40%（从高点）**
   - 新总资产: ¥675,000 × 0.6 = ¥405,000
   - NAV = 405,000/450,000 = 0.9
   - **各投资者盈亏**:
     - 张三: 200,000 × 0.9 - 200,000 = **-¥20,000** (-10%)
     - 李四: 150,000 × 0.9 - 150,000 = **-¥15,000** (-10%)
     - 王五: 100,000 × 0.9 - 100,000 = **-¥10,000** (-10%)

4. **继续下跌至NAV=0.6（总跌幅60%）**
   - 新总资产: 450,000 × 0.6 = ¥270,000
   - **各投资者盈亏**:
     - 张三: 200,000 × 0.6 - 200,000 = **-¥80,000** (-40%)
     - 李四: 150,000 × 0.6 - 150,000 = **-¥60,000** (-40%)
     - 王五: 100,000 × 0.6 - 100,000 = **-¥40,000** (-40%)

### Phase 3: 亏损时追加投资（定投策略）
5. **张三在低点追加投资**
   - 追加金额: ¥100,000
   - 当前NAV: 0.6
   - **获得份额**: 166,666.67份
   - **张三总持仓**:
     - 份额: 366,666.67份
     - 总投入: ¥300,000
     - 当前价值: 366,666.67 × 0.6 = ¥220,000
     - 盈亏: -¥80,000（成本摊薄效应）

6. **李四在更低点追加**
   - NAV跌至0.5时，李四追加¥100,000
   - **获得份额**: 200,000份
   - **李四总持仓**:
     - 份额: 350,000份
     - 总投入: ¥250,000
     - 加权平均成本: ¥250,000/350,000 = 0.714

### Phase 4: 回本过程
7. **NAV回升到0.8（未回本）**
   - 张三资产: 366,666.67 × 0.8 = ¥293,333（仍亏¥6,667）
   - 李四资产: 350,000 × 0.8 = ¥280,000（仍亏¥30,000）
   - **验证**: 李四成本价较高，更难回本

8. **NAV回升到0.9（张三回本）**
   - 张三资产: 366,666.67 × 0.9 = ¥330,000 > 300,000 ✅
   - 李四资产: 350,000 × 0.9 = ¥315,000 > 250,000 ✅
   - 王五资产: 100,000 × 0.9 = ¥90,000 < 100,000 ❌
   - **验证**: 定投者先回本，一次性投入者后回本

9. **NAV回升到1.0（全部回本并盈利）**
   - 全部投资者实现盈利

### Phase 5: 亏损时赎回测试
10. **NAV=0.7时张三部分赎回**
    - 赎回50,000份
    - **赎回金额**: 50,000 × 0.7 = ¥35,000
    - 实际亏损已兑现（相比成本价0.82）

11. **NAV=0.5时李四全部赎回（割肉）**
    - 全部350,000份赎回
    - **赎回金额**: 350,000 × 0.5 = ¥175,000
    - 实际亏损: ¥250,000 - ¥175,000 = **¥75,000**
    - 验证: 李四从投资者列表移除或标记为已清空

### Phase 6: 极端情况
12. **NAV跌至0.1（90%亏损）**
    - 验证系统正常处理极端低价
    - 剩余投资者资产计算正确

13. **最终回升到NAV=2.0（翻倍）**
    - 坚持到底的投资者获得高额回报
    - 验证回本点计算正确

## 关键验证点
- [ ] 下跌时盈亏计算正确
- [ ] 定投策略成本摊薄计算正确
- [ ] 不同成本价投资者回本时间不同
- [ ] 亏损时赎回实际亏损兑现
- [ ] 极端低价（NAV接近0）系统正常
- [ ] 加权平均成本计算准确

## 涉及后端能力

- 净值更新（多次，涨跌）
- 赎回（部分/全部，亏损状态）
- 盈亏计算
