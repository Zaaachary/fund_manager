# 测试样本 01: 基金完整生命周期与多投资者收益分配

## 测试目的
验证基金从创建到终止的完整生命周期，以及多个投资者在不同时间点进入时的收益分配是否正确（先入者获益更多）。

## 测试数据
- 基金名称: "成长一号"
- 成立日期: 2026-01-01
- 初始 NAV: 1.0

## 测试步骤

### Phase 1: 基金创建与早期投资者
1. **创建基金**
   - 基金名称: "成长一号"
   - 成立日期: 2026-01-01
   - 验证: 基金创建成功，初始NAV=1.0，总资产=0

2. **添加投资者A（早期）**
   - 投资者名称: "张三"
   - 加入日期: 2026-01-01
   - 验证: 投资者创建成功，份额=0，资产=0

3. **投资者A首次申购**
   - 申购金额: ¥100,000
   - 日期: 2026-01-01
   - 当前NAV: 1.0
   - **预期结果**: 获得100,000份
   - 验证: 张三份额=100,000，基金总份额=100,000，总资产=100,000

### Phase 2: 基金上涨，中期投资者进入
4. **第1次净值更新（上涨20%）**
   - 更新日期: 2026-02-01
   - 新总资产: ¥120,000
   - **预期计算**: NAV = 120,000/100,000 = 1.2
   - 验证: NAV=1.2，张三资产=120,000（浮盈20,000）

5. **添加投资者B（中期）**
   - 投资者名称: "李四"
   - 加入日期: 2026-02-01

6. **投资者B申购**
   - 申购金额: ¥100,000
   - 日期: 2026-02-01
   - 当前NAV: 1.2
   - **预期结果**: 获得83,333.33份（100,000/1.2）
   - 验证: 李四份额≈83,333.33，基金总份额≈183,333.33

### Phase 3: 继续上涨，晚期投资者进入
7. **第2次净值更新（再涨25%）**
   - 更新日期: 2026-03-01
   - 新总资产: ¥150,000
   - **预期计算**: NAV = 150,000/183,333.33 ≈ 0.8182 → 等等，应该是150,000
   - 重新计算: NAV = 150,000/183,333.33 = 0.818? 不对，总资产应该增加
   - 正确逻辑: 外部资本注入或投资组合增值
   - 假设: 投资组合增值到¥240,000
   - NAV = 240,000/183,333.33 ≈ 1.3091

8. **添加投资者C（晚期）**
   - 投资者名称: "王五"
   - 加入日期: 2026-03-01

9. **投资者C申购**
   - 申购金额: ¥100,000
   - 日期: 2026-03-01
   - 当前NAV: 1.3091
   - **预期结果**: 获得约76,388份

### Phase 4: 验证收益分配
10. **查看各投资者盈亏**
    - 张三: 100,000份 × 1.3091 ≈ ¥130,910（投入100,000，**盈利30,910**）
    - 李四: 83,333.33份 × 1.3091 ≈ ¥109,091（投入100,000，**盈利9,091**）
    - 王五: 76,388份 × 1.3091 ≈ ¥100,000（投入100,000，**盈利0**）
    - **关键验证**: 先入者张三盈利最多，后入者王五盈利最少

### Phase 5: 部分赎回与继续投资
11. **投资者A部分赎回**
    - 赎回份额: 30,000份
    - 日期: 2026-04-01
    - 当前NAV: 1.3091
    - **预期赎回金额**: 30,000 × 1.3091 ≈ ¥39,273
    - 验证: 张三剩余份额70,000，已落袋盈利

12. **投资者A追加申购**
    - 申购金额: ¥50,000
    - 日期: 2026-04-01
    - 当前NAV: 1.3091
    - **预期获得份额**: 38,194份
    - 验证: 张三总份额=108,194份（成本均价发生变化）

### Phase 6: 基金清算模拟
13. **最终净值更新**
    - 日期: 2026-06-01
    - 总资产: ¥400,000
    - 计算最终各投资者资产价值

14. **所有投资者全部赎回**
    - 张三全部赎回
    - 李四全部赎回
    - 王五全部赎回
    - 验证基金总份额归零，总资产归零

## 关键验证点
- [ ] 基金创建时NAV正确初始化为1.0
- [ ] 早期投资者（张三）享受最多涨幅，盈利最大
- [ ] 中期投资者（李四）享受部分涨幅，盈利中等
- [ ] 晚期投资者（王五）享受最少涨幅，盈利最小
- [ ] 部分赎回后份额计算正确
- [ ] 追加申购后成本均价重新计算
- [ ] 所有操作都有完整的历史记录

- 申购（按NAV计算份额）
- 净值更新（触发NAV重新计算）

- 操作历史查询
