# 测试样本 02: 复杂份额转账场景

## 测试目的
验证投资者之间份额转账的多种场景，包括部分转账、多次转账、转账后的净值变化等。

## 测试数据
- 基金名称: "转账测试基金"
- 成立日期: 2026-01-01
- 投资者: A(Alice), B(Bob), C(Carol), D(David)

## 测试步骤

### Phase 1: 初始建仓
1. **创建基金并初始化**
   - 基金名称: "转账测试基金"
   - 添加投资者A、B、C、D

2. **初始申购建立持仓**
   - Alice申购: ¥100,000 → 获得100,000份（NAV=1.0）
   - Bob申购: ¥50,000 → 获得50,000份
   - Carol申购: ¥80,000 → 获得80,000份
   - David申购: ¥30,000 → 获得30,000份
   - **验证**: 基金总份额=260,000份

3. **净值上涨到1.5**
   - 更新总资产到¥390,000
   - NAV = 390,000/260,000 = 1.5
   - **各投资者当前价值**:
     - Alice: 100,000 × 1.5 = ¥150,000
     - Bob: 50,000 × 1.5 = ¥75,000
     - Carol: 80,000 × 1.5 = ¥120,000
     - David: 30,000 × 1.5 = ¥45,000

### Phase 2: 单次转账场景
4. **Alice转部分份额给Bob**
   - 转账份额: 20,000份
   - **Alice剩余**: 100,000 - 20,000 = 80,000份
   - **Bob获得**: 50,000 + 20,000 = 70,000份
   - 验证转账后:
     - Alice资产: 80,000 × 1.5 = ¥120,000
     - Bob资产: 70,000 × 1.5 = ¥105,000
     - 基金总份额仍为260,000份（不变）

5. **Carol按金额转账给David**
   - 转账金额: ¥30,000
   - 当前NAV: 1.5
   - **转账份额**: 30,000/1.5 = 20,000份
   - **Carol剩余**: 80,000 - 20,000 = 60,000份
   - **David获得**: 30,000 + 20,000 = 50,000份
   - 验证: Carol资产=¥90,000, David资产=¥75,000

### Phase 3: 多次转账链条
6. **创建转账链条: A→B→C→D**
   - Alice转10,000份给Bob
   - Bob转15,000份给Carol（包含刚收到的）
   - Carol转25,000份给David
   - **验证每次转账后**:
     - 发送方份额减少
     - 接收方份额增加
     - 基金总份额始终不变

### Phase 4: 净值变化后转账
7. **净值更新到2.0**
   - 总资产: 260,000份 × 2.0 = ¥520,000
   - 各投资者资产翻倍

8. **高价时转账**
   - Alice转5,000份给Carol
   - **转账价值**: 5,000 × 2.0 = ¥10,000
   - 验证: NAV变化不影响转账逻辑，仍按份额计算

### Phase 5: 边界情况测试
9. **尝试超额转账（应失败）**
   - David尝试转出60,000份（实际只有50,000份）
   - **预期**: 系统拒绝，提示份额不足

10. **尝试转给不存在的投资者（应失败）**
    - 尝试转账给投资者E（未创建）
    - **预期**: 系统拒绝，提示投资者不存在

11. **转账全部份额**
    - David转出全部50,000份给Alice
    - **David份额**: 归零
    - **验证**: David仍在投资者列表，份额为0

### Phase 6: 转账后赎回验证
12. **转账后部分赎回**
    - Alice赎回20,000份（当前NAV=2.0）
    - **赎回金额**: ¥40,000
    - 验证: 不影响之前的转账记录

13. **最终全部赎回清算**
    - 所有投资者全部赎回
    - 验证基金归零

## 关键验证点
- [ ] 单次转账后双方份额正确更新
- [ ] 多次转账链条处理正确
- [ ] 按金额转账自动计算份额
- [ ] 净值变化后转账价值相应变化
- [ ] 超额转账被正确拒绝
- [ ] 转账给不存在投资者被拒绝
- [ ] 全部份额转出后投资者仍存在（份额为0）
- [ ] 转账操作有完整历史记录

## 涉及后端能力
- 投资者添加
- 申购
- 净值更新
- 转账（按份额/按金额）
- 赎回
- 操作历史
