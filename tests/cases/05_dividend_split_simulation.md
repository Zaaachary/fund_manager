# 测试样本 05: 复杂业务场景 - 基金分红与拆分模拟

## 测试目的
通过净值更新机制模拟基金分红和份额拆分场景，验证系统在复杂业务逻辑下的正确性。

## 测试数据
- 基金名称: "分红拆分模拟基金"
- 投资者: A, B, C三位长期持有者

## 测试步骤

### Phase 1: 长期持有与积累
1. **初始建仓（模拟IPO）**
   - 投资者A申购: ¥500,000 → 500,000份
   - 投资者B申购: ¥300,000 → 300,000份
   - 投资者C申购: ¥200,000 → 200,000份
   - 总份额: 1,000,000份，NAV=1.0

2. **长期增长阶段**
   - 经历6次净值更新，NAV从1.0增长到2.5
   - 总资产从¥1,000,000增长到¥2,500,000
   - 各投资者盈利150%

### Phase 2: 模拟分红（通过净值调整实现）
3. **现金分红场景模拟**
   - 分红前NAV: 2.5
   - 每份分红: ¥0.5
   - **分红后NAV**: 2.5 - 0.5 = 2.0
   - 更新总资产: 1,000,000份 × 2.0 = ¥2,000,000
   - **各投资者分红金额**:
     - A: 500,000 × 0.5 = ¥250,000
     - B: 300,000 × 0.5 = ¥150,000
     - C: 200,000 × 0.5 = ¥100,000
   - **验证**: 分红后份额不变，NAV下降，资产价值减少分红额

4. **分红后再投资**
   - 投资者A将分红全部再投资: ¥250,000 / NAV2.0 = 125,000份
   - A新份额: 625,000份
   - 验证: 再投资后份额增加，成本价变化

### Phase 3: 模拟份额拆分
5. **2:1份额拆分场景模拟**
   - 拆分前: NAV=2.0, A有625,000份
   - 拆分后目标NAV=1.0（便于交易）
   - **实际操作**: 净值更新时调整
   - 新总资产: ¥2,000,000
   - 新NAV: 1.0
   - 新总份额: 2,000,000份（虚拟增加）
   - 各投资者份额翻倍:
     - A: 625,000 → 1,250,000份
     - B: 300,000 → 600,000份
     - C: 200,000 → 400,000份
   - **验证**: 份额翻倍，NAV减半，总价值不变

### Phase 4: 拆分后交易
6. **拆分后申购**
   - 新投资者D在NAV=1.0时申购¥100,000 → 100,000份
   - 验证: 新投资者以低价进入

7. **拆分后赎回**
   - 投资者B赎回200,000份（拆分后份额的1/3）
   - 赎回金额: 200,000 × 1.0 = ¥200,000
   - 验证: 赎回计算正确

### Phase 5: 反向拆分（合并）模拟
8. **1:2份额合并场景**
   - 合并前NAV=0.5（股价过低）
   - 目标NAV=1.0
   - **操作**: 净值更新
   - 新NAV=1.0
   - 各投资者份额减半（虚拟减少）:
     - A: 1,250,000 → 625,000份
     - C: 400,000 → 200,000份
   - **验证**: 份额减半，NAV翻倍，总价值不变

### Phase 6: 复杂历史场景验证
9. **完整历史回测**
   - 查看投资者A的完整历史:
     - 初始申购
     - 多次净值更新（涨）
     - 分红
     - 再投资
     - 拆分
     - 合并
   - **验证**: 每次操作后份额、资产、成本价计算正确
   - 验证最终盈亏计算准确

10. **成本价计算验证**
    - 经过多次分红再投资，成本价应动态调整
    - 验证: 成本价 = 总投入 / 当前份额
    - 投资者A最终成本价计算验证

### Phase 7: 税务场景模拟
11. **持有期计算**
    - 记录每笔申购的日期
    - 计算不同批次的持有期
    - 模拟: 赎回时优先赎回早期份额（FIFO）
    - **验证**: 系统记录每笔操作时间

12. **收益分类模拟**
    - 分红收益 vs 资本利得
    - 通过操作类型区分
    - 验证操作记录完整性

## 关键验证点
- [ ] 分红机制通过净值更新正确模拟
- [ ] 分红后份额不变，NAV下降
- [ ] 份额拆分后份额翻倍，NAV减半
- [ ] 份额合并后份额减半，NAV翻倍
- [ ] 每次调整后总价值保持不变
- [ ] 复杂历史后成本价计算正确
- [ ] 再投资后份额和成本正确更新
- [ ] 操作历史完整记录所有变动

## 涉及后端能力
- 净值更新（核心机制）
- 申购（多次，不同时间点）
- 赎回
- 投资者查询（查看完整历史）
- 操作历史查询
- 图表数据（展示净值变化）

## 备注
虽然系统没有直接的分红/拆分API，但可以通过**净值更新机制**配合**操作记录**来完整模拟这些场景。
